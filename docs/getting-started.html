Getting Started

Build & Run OpenIntentOS

OpenIntentOS compiles to a single ~10 MB binary. All you need is the Rust toolchain and at least one LLM API key.

Prerequisites

Rust
1.80+ (2024 edition)
OS
macOS · Linux · Windows
RAM
512 MB minimum
LLM key
At least one provider

Install Rust

# Install rustup (skip if already installed)
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh

# Verify
rustc --version   # should be ≥ 1.80
cargo --version

Clone & Build

# Clone the repository
git clone https://github.com/openintent/OpenIntentOS
cd OpenIntentOS

# Release build (optimised, ~10 MB binary)
cargo build --release

# The binary ends up at:
./target/release/openintent-cli

Build time is typically 2–4 minutes on first compile (compiles all crates). Incremental rebuilds are under 10 seconds.

Environment Variables

Set these before running. Only TELEGRAM_BOT_TOKEN and one LLM key are strictly required.

# Example .env (never commit this file)
export TELEGRAM_BOT_TOKEN="7123456789:AAFxxxx"
export OPENAI_API_KEY="sk-proj-xxxx"
export NVIDIA_API_KEY="nvapi-xxxx"
export GOOGLE_API_KEY="AIzaSyxxxx"
export DEEPSEEK_API_KEY="sk-xxxx"
export GROQ_API_KEY="gsk_xxxx"

Configuration File

The main config lives at config/default.toml. The bot reads it at startup and hot-reloads changes automatically.

# config/default.toml (key sections)

[provider]
name = "openai"
base_url = "https://api.openai.com/v1"
model = "chatgpt-pro"

[bot]
history_window = 20         # messages of context kept per chat
show_token_usage = false    # toggle with /tokens on|off in chat
simple_query_threshold = 120 # chars; short queries use cheap model

Run the Telegram Bot

# Foreground (logs to stdout)
cargo run --release --bin openintent-cli -- bot

# Background with logging to file
cargo run --release --bin openintent-cli -- bot > /tmp/bot.log 2>&1 &

# Check it's alive
tail -f /tmp/bot.log
Expected startup output
INFO starting Telegram bot gateway INFO database pragmas applied (WAL, mmap 256MiB, cache 62MiB) INFO adapters online: filesystem shell web_search browser feishu calendar email github INFO skills loaded count=5 INFO Bot: @OpenIntentbot · Provider: OpenAI · Model: chatgpt-pro INFO Bot is running. Send messages to @OpenIntentbot on Telegram.

OpenAI

Default provider. Set OPENAI_API_KEY and the bot uses it automatically. To use a specific model:

export OPENAI_API_KEY="sk-proj-xxxx"
# In config/default.toml:
[provider]
model = "chatgpt-pro"  # or gpt-4o, o3-mini, etc.

NVIDIA NIM (Free Tier)

Sign up at build.nvidia.com for a free API key. Models available depend on your account — check the NIM catalog to see what's enabled.

export NVIDIA_API_KEY="nvapi-xxxx"

The failover chain automatically uses NVIDIA NIM as the first fallback. You can also manually switch: /model nvidia

Google Gemini

Get a key at aistudio.google.com. Gemini is used as the second cascade fallback.

export GOOGLE_API_KEY="AIzaSyxxxx"

DeepSeek

export DEEPSEEK_API_KEY="sk-xxxx"

DeepSeek Reasoner (deepseek-reasoner / alias r1) is excellent for step-by-step reasoning tasks.

Groq

Groq offers the fastest inference latency of any cloud provider — useful for simple queries.

export GROQ_API_KEY="gsk_xxxx"

Ollama (Local)

Install Ollama and pull a model. The bot connects to it on localhost:11434.

# Install and pull
ollama pull llama3.2

# config/default.toml
[provider.ollama]
base_url = "http://localhost:11434/v1"
model = "llama3.2"

Browser Adapter (CDP)

Launch Chrome with remote debugging enabled before starting the bot:

# macOS
/Applications/Google\ Chrome.app/Contents/MacOS/Google\ Chrome \
  --remote-debugging-port=9222 --no-first-run &

# Linux
google-chrome --remote-debugging-port=9222 --no-first-run &

The bot detects it automatically at startup: DevTools endpoint already reachable.

Email Adapter

# config/default.toml
[adapters.email]
imap_host = "imap.gmail.com"
imap_port = 993
smtp_host = "smtp.gmail.com"
smtp_port = 587
username  = "you@gmail.com"
# Password/OAuth token stored in encrypted vault

GitHub Token

Setting GITHUB_TOKEN enables the evolution self-repair system: the agent can read its own issues and PRs to plan fixes.

export GITHUB_TOKEN="ghp_xxxx"
